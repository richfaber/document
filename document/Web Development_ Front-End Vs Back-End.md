# (의역) Web Development: Front-End Vs Back-End

@출처 : http://techtiquk.blogspot.com/2018/08/decoding-web-development-front-end-vs.html?m=1

많은 웹사이트와 웹개발자 들이 생겨났음에도 불구하고, 많은 사람들이 실제로 [프론트엔드 개발](https://www.techtiq.co.uk/full-stack-expert) 과 [백엔드 개발](https://www.techtiq.co.uk/full-stack-expert) 의 차이점을 모릅니다. 

서로 어떤 차이점이 있을까요?
  
## 프론트엔드 개발자

웹개발의 프론트엔드는 글꼴 및 색상, 드롭 다운 메뉴 및 슬라이더, 등의 사용자가 상호 작용하는 모든 것을 제작 합니다. 

컴퓨터 브라우저가 이해하는 HTML, CSS 및 JavaScript 조합을 사용하여, 웹 사이트의 **사용자 경험(UX)**을 비롯한 모든 코드와 아키텍처는  **프런트 엔드 웹 개발자** 가 코딩 합니다.  

![Front End Development ](http://iynsoft.com/assets/images/front-end.gif)](http://iynsoft.com/assets/images/front-end.gif)

### Languages, Skills and Tools  

웹사용자가 웹사이트에 접속해서 좋은 사용자경험(UX) 을 얻게 하기 위해서, 프론트엔드 개발자가 알아야 하는 세가지 주요 언어는  HTML, CSS 및 자바스크립트로, 이 세 가지 주요 언어는 필수로 능숙 해야 합니다.

또한 모든 장치에서 컨텐츠가 잘 보이도록 하기 위한 프레임 워크 (예 : Bootstrap, Foundation, Backbone, AngularJS 및 EmberJS) 및 UI 동작을 제어할 수 있는 코드를 작성하기 위해 jQuery 와 같은 라이브러리를 잘 알고 있어야 더 많은 형태의 웹사이트를 빠르고 편리하게 제작을 할 수 있습니다. 

또한 서버 데이터를 다운로드 하여 페이지를 동적으로 로드 할 수 있게 해주는 Javascript 와 함께 사용할 수 있는 기술인 **Ajax 사용 경험** 또한 프론트 엔드 개발자가 가지고 있어야만 합니다. 
많은 회사들이 프론트엔드 개발자 구인공고에 Html,Css,Javascript,Ajax 등을 기본으로 두고 있는 이유 이기도 합니다.

프런트 엔드 개발자는 디자이너 또는 사용자 경험 분석가(UX) 와 긴밀히 협력하여 개발에서 제작에 이르기 까지 이러한 언어와 기술을 사용하여 와이어 프레임을 제작합니다. 잘하는 프런트 엔드 개발자는 사용자 환경의 특정 문제를 정확하게 지적하고 코드화 된 솔루션을 제공하여 디자인을 향상 시킬 수 있고, 이렇게 잘 만들어진 코드는 유지보수를 편리하게 만들어 줍니다.

## 백엔드 개발자
  
웹 사이트의 백엔드개발자는 서버, 응용 프로그램 및 데이터베이스를 관리합니다. 백 엔드 개발자는 처음부터 웹 사이트의 사용자가 접속할 수 있도록 하는 기반이 되는 기술을 구축하고 유지 관리합니다.

사용자가 구글에 접속을 한다고 하면, Google 서버가 사용자의 컴퓨터 또는 휴대 기기로 브라우저가 해석할 수 있는 정보를 전송하여, 사용자는 화면을 볼 수 있게 됩니다. 

또한 사용자가 웹사이트에 접속하여 어떤 직업을 신청 하거나, 이력서를 업로드 하는 경우 개인 정보를 기록하는 것을 백엔드 개발자가 전송받아서 서버에 저장하거나, DB 에 기록합니다.
  
![back End Development ](https://ultimez.com/blog/wp-content/uploads/2016/09/gif-animation.gif)](https://ultimez.com/blog/wp-content/uploads/2016/09/gif-animation.gif)

### Languages, Skills and Tools

백엔드 개발자는 서버 측 언어 (예 : Ruby, PHP, Python, Java 및 .Net)를 사용하여 응용 프로그램을 구축하고 Oracle, MySQL 및 SQL Server 와 같은 도구를 사용하여 데이터를 찾고, 저장하거나 변경하고 이를 제공 합니다. 

이렇게 제공된 데이터는 프론트 엔드 코드로 사용자 에게 다시 알려줍니다.

프런트엔드 작업과 백엔드 작업을 모두 수행하는 개발자는 종종 **"풀 스택 개발자"** 라고 합니다. 
**풀스택 개발자** 라고 자칭하는 개발자들 중에는, 한쪽으로 치우친 경우가 많고, 진정한 풀스택 개발자라고 하면, 상대적으로 찾기가 어렵습니다.

개발자의 기술적 역할을 정의 할 때 많은 부분이 두리뭉실 하게 나눕니다. 대체적으로 회사는 개발자 고용시 이런 기술적 역할의 정의 없이 약간 혼합된 기술 집합과 성격을 찾습니다. 

하지만 프론트엔드와 백엔드가 하는 역할을 이해하게 된다면, 좀더 정확하고 빠르게 인재를 구할 수 있을 것이라고 생각합니다.
또한 **풀스택 개발자** 는 아주 많은 기술적 지식을 공부 했고, 글로만 알고 있는 것이 아니라 실제 능숙하게 응용할 수 있을 때 **풀스택 개발자** 라고 정의할 수 있기 때문에, 많은 비용을 지불해야 한다는 것도 이해 해야만 합니다.
